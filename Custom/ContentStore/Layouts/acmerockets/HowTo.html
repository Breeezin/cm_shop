How to Create a Layout

Adding a new layout

When a layout is first created, it should be named lyt_layoutname.cfm and the stylesheet for it should be named sty_stylesheetname.cfm.

The layout then needs to be included in the layouts.txt file so that it can be selected from the content manager. To do this just enter a new line in the file like so: 

	layoutname:stylesheetname:displayname
Displayname is what the layout will appear as in the content manager selector.

Using the default layout as an example, the layout file is called lyt_default.cfm, the style sheet file is called sty_main.cfm and the layout is displayed as "Default Layout" in the content manager, so the line in Layouts.txt would be: 

	default:main:Default Layout

Inserting layout elements

The various elements of the layout can be inserted simply by typing [ElementName] into the layout file at the position you wish it to appear. The following layout elements are defined:

NameDescription
Content The full content for this page.
Description The description for this page, either the website default or the description specified for this page. This should be inserted in the "description" meta information.
Menu See the section below for information on inserting a menu element.
Keywords The keywords for this page, either the website default or the keywords specified for this page. This should be inserted in the "keywords" meta information.
SiteName The name of the site. e.g. "Carnivores"
Title The text name of the currently viewed page. e.g. "Contact Us"
TitleImage The title image of the currently viewed page, or the text name of the page if no title image exists.

Menus

Defining a menu

A menu may be defined by enclosing it's attributes in [MENU]...[/MENU] or alternatively by inserting the menu definition in an image tag's ALT field. For example:

  • <IMG SRC="foo" ALT="MENU:Attribute:Value:Attribute:Value">
  • [MENU]Attribute:Value:Attribute:Value[/MENU]

Attributes

There are two types of menus; Standard and Footer. Standard is a vertical or horizontal menu which may use images, mouseovers and drop down menus. Footer is a text-based horizontal menu with separators that can be defined to separate the menu items.

Shared Attributes

The following attributes can be applied to both types of menus.

NameDefaultDescription
Type Type of menu. Standard or Footer.
Root /index.cfm The menu will display the sub-assets of this asset.
LinkClass See description The class to apply to text-links in the menu. The default value for Footer menus is "FooterLink". The default value for Standard menus is "MenuLink".
LowerCase No If set to Yes, menu items will appear in lowercase.
UpperCase No If set to Yes, menu items will appear in uppercase.

Standard Menu Attributes

The following attributes can only be applied to menus with type set to Standard.

NameDefaultDescription
Orientation Vertical The orientation of the menu, either Vertical or Horizontal.
Separator Any HTML code to insert between menu items. Will be automatically enclosed in appropriate table cell tags.
TableClass MenuTable The name of the class to apply to the <TABLE>...</TABLE> surrounding the menu.
FirstCellClass MenuFirstCell The name of the class to apply to the <TD>...</TD> surrounding the first item in the menu.
OtherCellClass MenuOtherCell The name of the class to apply to the <TD>...</TD> surrounding the all items except the first item in the menu.
SeparatorOtherClass MenuSeparatorCell The name of the class to apply to the <TD>...</TD> surrounding the menu separator cells.
CellPadding 0 CellPadding value for the menu TABLE. (Mozilla seems to ignore cell padding if it is set in a stylesheet).
CellSpacing 0 CellSpacing value for the menu TABLE. (Mozilla seems to ignore cell spacing if it is set in a stylesheet).
DDWidth 120 The width of the drop down menus.
DDRowHeight 10 The height of each row in the drop down menus.
DDOffsetX 100 (Vertical menu)
0 (Horizontal menu)		 How far to position the drop down menus from the upper left corner of the parent menu item.
DDOffsetY 0 (Vertical menu)
20 (Horizontal menu)		 How far to position the drop down menus from the upper left corner of the parent menu item.
DDFontColor #000000 The font color for items in drop down menus.

Footer Menu Attributes

The following attributes can only be applied to menus with type set to Footer.

NameDefaultDescription
Separator | The text to insert between menu items. May include HTML code.
LinksPerRow 1024 Maxmium number of items per row. Inserts a <BR> tag instead of a separator after the last item in each row.
ShowAllAssets No If set to Yes, the footer menu will display all assets even if they are not set to appears in menus.

An Example Layout

<HTML>
<HEAD>
  <TITLE>[SiteName] - [Title]</TITLE>
  <META NAME="keywords"    CONTENT="[Keywords]">
  <META NAME="description" CONTENT="[Description]">
  <LINK REL="STYLESHEET" TYPE="text/css" HREF="sty_main.css" MEDIA="screen">
</HEAD>
<BODY>
  <P><IMG SRC="foo" ALT="MENU:Type:Standard:Orientation:Horizontal:UpperCase:Yes"></P>
  <P><H3>[TitleImage]</H3>
    [Content]
  </P>
  <P ALIGN="CENTER">[MENU]Type:Footer:Uppercase:Yes[/MENU]</P>
</BODY>
</HTML>
